# 개인정보 처리 가이드 (챗봇)

이 문서는 Soft Story 상담 챗봇을 웹사이트에 붙일 때 필요한 **개인정보 처리 관련 정리**입니다.
법률 자문이 아니라 개인정보보호위원회 가이드 + 실무 기준에 따른 정리이며, 최종 문안은
필요 시 전문가 확인을 권장합니다.

핵심 한 줄:
> **"스크립트를 어떻게 붙이느냐(위젯 vs 통합)"와 "개인정보처리방침"은 별개 문제다.**
> **통합해도 대화는 OpenAI(미국)로 나가므로, 처리방침 갱신은 어느 쪽이든 필요하다.**

---

## 1. 데이터가 어디로 가는가 (흐름)

```
방문자 입력
   │  (채팅창에 질문 입력)
   ▼
우리 Vercel 백엔드  (/api/chat)        ← 대화를 저장하지 않음 (stateless, 메모리 처리 후 폐기)
   │
   ▼
OpenAI API (미국)                       ← 답변 생성. 여기서 "국외이전"이 발생
   │
   ▼
답변을 방문자 화면으로 스트리밍
```

**기억할 사실 3가지**

1. **우리 앱은 대화를 저장하지 않는다.** 메모리에서 처리 후 버린다 → 우리 쪽 DB에 쌓이는 대화 로그가 없다. (보호에 유리)
2. **하지만 대화는 OpenAI(미국)로 나간다.** 방문자가 이름·전화·이메일을 적으면 그 개인정보가 국외로 이전된다.
3. **OpenAI API는 ChatGPT 일반 버전과 다르다.**
   - API 데이터는 **모델 학습에 사용되지 않는다(기본값).**
   - 악용 감시용으로 **최대 30일 보관 후 삭제.**
   - 더 강한 보호가 필요하면 **표준 DPA** 및 기업용 **무보관(Zero Data Retention)** 옵션 제공.

---

## 2. 법적으로 건드리는 지점 2가지

| 구분 | 내용 | 우리에게 적용 |
|---|---|---|
| **개인정보 처리위탁** | 외부 API에 고객 개인정보를 전송하면 위탁에 해당. 수탁자를 처리방침에 공개·감독해야 함 | OpenAI(답변 생성), Vercel(호스팅)이 수탁자 |
| **개인정보 국외이전** | 원칙 금지. 단, **처리방침에 공개**했거나 **별도 동의**가 있으면 예외 허용 | 대화가 미국(OpenAI)으로 가므로 처리방침에 국외이전 항목 공개 필요 |

---

## 3. 위험을 거의 없애는 가장 싼 방법 — PII 입력 차단 안내

> 채팅창에 **"개인정보(이름·연락처 등)는 입력하지 마세요"** 안내 한 줄을 띄운다.

논리: 방문자가 개인정보를 안 적으면 국외로 나가는 건 "서비스 질문"뿐 → **개인정보 국외이전 자체가 성립하지 않는다.** 처리방침 부담도 크게 줄어든다. 우리 봇은 마케팅·서비스 Q&A용이므로 용도와도 부합.

**누출 위험 3중 방어**

1. PII 입력 금지 안내 (가장 중요, 코드로 추가 가능)
2. 우리 앱 무저장 (이미 적용됨)
3. OpenAI 30일 후 삭제 + 학습 미사용 (정책상 보장)

### 채팅 UI 고지 문구 (복사용)

웰컴 화면 또는 입력창 하단에 작게 노출:

```
※ 본 챗봇은 서비스 안내용입니다. 이름·연락처 등 개인정보는 입력하지 마세요.
   상담·견적은 문의 폼을 이용해 주세요.
```

영문 병기가 필요하면:

```
This chatbot is for service information only. Please do not enter personal data
(name, contact, etc.). For quotes/inquiries, please use the contact form.
```

당사는 웹사이트 방문자의 서비스 문의 응대를 위해 AI 상담 챗봇을 운영합니다. 챗봇 이용 시
개인정보가 아래와 같이 처리됩니다.

**1) 처리 항목 및 목적**
- 처리 항목: 방문자가 채팅창에 입력한 대화 내용
- 처리 목적: 서비스 관련 질문에 대한 자동 응답 제공
- 당사는 챗봇 대화 내용을 별도로 저장하지 않으며, 응답 생성을 위한 처리 후 폐기합니다.

**2) 개인정보 처리위탁**

| 수탁자 | 위탁 업무 |
|---|---|
| OpenAI, L.L.C. | 입력 내용에 대한 AI 응답 생성 |
| Vercel Inc. | 챗봇 서비스 호스팅 |

당사는 위탁계약 시 개인정보가 안전하게 관리되도록 관련 사항을 규정하고 수탁자를 감독합니다.

**3) 개인정보의 국외 이전**

챗봇 응답 생성을 위해 입력 내용이 아래와 같이 국외로 이전될 수 있습니다.

| 항목 | 내용 |
|---|---|
| 이전받는 자 | OpenAI, L.L.C. |
| 이전 국가 | 미국 |
| 이전 일시 및 방법 | 챗봇 이용 시점에 정보통신망을 통한 전송 |
| 이전 항목 | 방문자가 채팅창에 입력한 대화 내용 |
| 이전받는 자의 이용 목적 | AI 응답 생성 |
| 보유·이용 기간 | OpenAI 정책에 따라 최대 30일 보관 후 삭제(모델 학습에는 사용되지 않음). 당사는 별도 보관하지 않음 |

**4) 이용자 유의사항**

챗봇은 서비스 안내 목적이며, 이름·연락처 등 개인정보를 입력하지 않도록 안내하고 있습니다.
구체적인 상담·견적 문의는 별도의 문의 양식을 이용해 주시기 바랍니다.

**5) 시행일**
- 본 방침의 챗봇 관련 사항은 〔YYYY년 MM월 DD일〕부터 시행합니다.

## 5. 임베드(붙이는 방식)와 보안 — 참고

| | 외부 위젯 `<script>` 한 줄 | 한 레포로 통합 / 우리 도메인 서빙 |
|---|---|---|
| 스크립트 신뢰 | 외부 도메인 스크립트가 host 페이지에 접근 가능 | 같은 도메인 → 우려 해소 |
| 도메인 노출 | `*.vercel.app` 노출 | `softstorycorp.com` 일관 |
| 관리 | 레포 2개·배포 2번 | 코드 1개·배포 1번 |
| OpenAI 데이터 흐름 | OpenAI로 감 | **동일하게 OpenAI로 감 (변화 없음)** |

- 현재 위젯은 **버튼 + iframe** 구조라, 채팅 UI가 host 페이지(예: contact 폼) 데이터를
  들여다볼 수 없다 → 보안상 권장 패턴([public/widget.js](../public/widget.js)).
- softstorycorp.com이 **Next.js면 통합이 깔끔**, 다른 스택이면 **chat.softstorycorp.com**
  서브도메인으로 우리 코드를 그대로 올리는 방식도 가능(둘 다 "우리 도메인"이라 신뢰 문제 해소).
- **단, 통합/서브도메인 어느 쪽도 OpenAI로의 데이터 이전은 그대로**이므로 §4 처리방침 갱신은 필요.
